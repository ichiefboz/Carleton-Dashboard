Welcome to the GitHub site for The Dashboard.

Check out the live site at sayleshill.xyz. This page discusses how to implement the database protocol if you'd like to include data in your own website. Keep in mind that hours can fluctuate between terms and breaks, so this data might not be fully accurate.

To get the current and upcoming statuses of buildings, send a JSON dictionary with categories (which you define) and building IDs (which are pre-defined in the database) to api.php. This dictionary goes in the buildings POST field.

Here are some examples of valid dictionaries:

{"Sayles-Hill Campus Center": ["sayles", "security", "security-office", "sayles-cafe", "bookstore", "post", "onecard", "ccce", "career", "sao", "info", "krlx"],
"Dining": ["burton", "ldc", "east-express", "sayles-cafe", "weitz-cafe", "dominos"],
"Laurence McKinley Gould Library": ["libe", "reference", "libe-it", "writing", "archives"],
"Academic Support": ["msc", "writing", "language"],
"Information Technology Services": ["its", "libe-it", "peps"],
"Recreation": ["rec", "wall-bouldering", "cowling", "cowling-pool", "west", "west-pool", "stadium"],
"Business and Administration": ["registrar", "business", "mail", "print"],
"Other": ["shac", "dacie-moses", "perlman", "idealab"]}
{"Buildings": ["sayles"]}

The site will then give you a JSON array back, sorted by your defined categories, with information for each building.
